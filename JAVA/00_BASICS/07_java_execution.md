# Execution of a Java Program

When a Java program runs, it goes through the following stages:

```
Source Code (.java)
       ↓
  [ Compiler (javac) ]
       ↓
  Byte Code (.class)
       ↓
  [ Interpreter (JVM) ]
       ↓
    Output
```

---

## 1. Compiler

The **compiler** (`javac`) is a software that takes the **source code (`.java` file)** as input.

- Checks for **syntactical mistakes** in the source code.
- Compiles (checks) the source code **top to bottom, all at once**.
- If a mistake is found, it throws a **Compile Time Error**.
- Displays **all errors at once** — does not stop at the first error.
- If there are **no errors**, it generates an intermediate file called **Byte Code (`.class` file)**.

**Command for compilation:**
```bash
javac FileName.java
```

### 📝 Note
- Compiler generates a new byte code file only if one does **not already exist** with the same class name.
- If a byte code file already exists with the same class name, the compiler **overwrites** it instead of creating a new one.

---

## 2. Byte Code

**Byte Code** is an intermediate code generated by the Java compiler (`javac`) after successfully compiling a `.java` file.

- It is **neither directly understandable by the programmer** nor by the machine/computer.
- It acts as a **middle layer** between source code and machine code.
- Instead of binary (0s and 1s), JVM instructions are written as readable keywords:

```
iload    → load an integer value
iadd     → add two integers
isub     → subtract two integers
imul     → multiply two integers
ireturn  → return an integer value
```

- File extension is **`.class`** and the filename is the same as the class name.
- Byte code is passed as **input to the Interpreter (JVM)**.

---

## 3. Interpreter (JVM)

The **Interpreter (JVM — Java Virtual Machine)** is a software that takes **byte code** as input.

- Reads and interprets the byte code **line by line**.
- Checks for **logical mistakes** during execution.
- If an error is found, it **immediately stops execution** and displays the error.
- Error given by the interpreter is known as a **Runtime Error**.
- Throws **only one error at a time** (stops at the first error encountered).

**Command for execution:**
```bash
java ClassName
```

### 📝 Note
> The reason Java is called **"Write Once, Run Anywhere (WORA)"** is because of byte code. The same `.class` file can run on any machine that has a JVM installed, regardless of the operating system.

---

## 4. Debugging

**Debugging** is the process of finding, analyzing, and fixing errors (bugs) in a program.

- Ensures the program runs correctly and produces the expected output.
- Debugging **compile time errors** is relatively easier since the compiler points them all out at once.
- Debugging **runtime errors** can be harder as they only appear during execution and stop the program mid-way.

---

## Compiler vs Interpreter — Comparison Table

| Feature | Compiler | Interpreter |
|---|---|---|
| **Input** | Source code (`.java`) | Byte code (`.class`) |
| **Converts** | High-level language → Byte code | Byte code → Machine code |
| **Checks for** | Syntactical mistakes | Logical mistakes |
| **Error type** | Compile Time Error | Runtime Error |
| **Error display** | All errors at once | One error at a time |
| **On error** | Stops and shows all errors | Stops at first error |
| **Speed** | Faster | Slower |
| **Output file** | Generates byte code (`.class`) | Does not generate any file |
| **Debugging** | Difficult | Easier |

---

## Types of Errors in Java

| Error Type | Thrown By | Cause | Example |
|---|---|---|---|
| **Compile Time Error** | Compiler | Syntax mistakes in source code | Missing semicolon, wrong keyword |
| **Runtime Error** | Interpreter (JVM) | Logical mistakes during execution | Dividing by zero, null pointer |
