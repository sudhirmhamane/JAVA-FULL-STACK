# Foreign Key

- It is the type of contraints we assign to perticular column in order to establish connection between two or multiple tables.
- Foreign key need to be unique.
- Foreign key can be null.
- Only a primary key can become a foreign key in another table.
- we can have any number of foriegn key in table.
- Establishes a relationship between **two tables**
- Refers to the **primary key** of another table
- Maintains **referential integrity**

Used when:
- Linking child table with parent table
