# Primary Key

- It is the type of contraint we assign to a perticular column which is used to identify records uniquely from the table.
- Primary key must be unique in nature.
- It must be not null.
- Only one primary key allowed or used in the Table.
- Uniquely identifies each record in a table
- Combination of **UNIQUE + NOT NULL**
- Only one primary key is allowed per table

Used when:
- Employee ID
- Student ID
- Account number
